# Technology Stack

**Analysis Date:** 2026-04-15

## Languages

**Primary:**
- Bash (5.x via Flox/Nix) — Main implementation language for the sandbox wrapper (`sandflox`), all test scripts, the Flox manifest hooks/profile (`manifest.toml`), generated enforcement scripts (`fs-filter.sh`, `entrypoint.sh`)
- Python 3 (3.6+ minimum, 3.11+ preferred for `tomllib`) — Inline policy parser embedded in `sandflox` and `manifest.toml` hook, `usercustomize.py` runtime monkey-patching for `builtins.open` and `ensurepip` blocking

**Secondary:**
- TOML — Declarative policy configuration (`policy.toml`) and Flox environment manifest (`.flox/env/manifest.toml`)
- Apple Seatbelt Profile Language (SBPL) — Generated macOS kernel sandbox profiles (`.flox/cache/sandflox/sandflox.sb`)

## Runtime

**Environment:**
- Flox 1.10+ — Reproducible Nix-based environment manager; provides the controlled `$FLOX_ENV/bin` namespace
- Nix (via Flox) — Package resolution backend; sandflox explicitly blocks direct Nix access
- macOS sandbox-exec — Kernel-level enforcement on Darwin via SBPL profiles
- Linux bubblewrap (bwrap) — Kernel-level enforcement on Linux via namespaces

**Package Manager:**
- Flox — Declarative package management via `.flox/env/manifest.toml`
- Lockfile: `.flox/env/manifest.lock` (present, JSON format, auto-generated by Flox)

## Frameworks

**Core:**
- Flox manifest system (schema 1.10.0) — Provides hook/profile lifecycle, package installation, environment variables, and cache directories

**Testing:**
- Custom bash test harness — Three hand-rolled test scripts using `ok()`/`bad()`/`skp()` pattern functions with pass/fail/skip counters
  - `test-policy.sh` — 29-test suite for v2 policy.toml + kernel enforcement
  - `test-sandbox.sh` — 35-test suite for shell-level enforcement (backward compat)
  - `verify-sandbox.sh` — Quick verification script for all enforcement layers

**Build/Dev:**
- None — No build step. The project is interpreted Bash + Python. `chmod +x sandflox` is the only setup required.

## Key Dependencies

**Critical (Flox-managed packages in `manifest.toml`):**
- `bash` — Shell runtime for sandbox wrapper and all scripts
- `python3` — TOML policy parser and `usercustomize.py` runtime enforcement
- `coreutils` — Core POSIX utilities (ls, cp, mv, mkdir, etc.)
- `curl` — HTTP client (conditionally removed from PATH when `network = "blocked"`)
- `git` — Version control (available in default and full profiles)
- `jq` — JSON processing
- `gnugrep` — grep/egrep/fgrep
- `gnused` — sed
- `gawk` — awk
- `findutils` — find/xargs
- `diffutils` — diff/cmp

**Infrastructure:**
- `sandbox-exec` (macOS system binary) — Apple Seatbelt sandbox enforcement; ships with macOS, not installed via Flox
- `bwrap` (bubblewrap, Linux) — Linux user-namespace sandbox; must be installed separately on Linux hosts

**Python Standard Library (no pip dependencies):**
- `tomllib` (Python 3.11+) — TOML parsing; falls back to inline minimal parser for Python 3.6-3.10
- `json` — Config serialization between Bash and Python
- `os`, `sys`, `re`, `shutil`, `types`, `importlib`, `builtins`, `socket`, `errno` — Various stdlib modules used in parser and enforcement scripts

## Configuration

**Environment:**
- `SANDFLOX_PROFILE` — Selects active profile (overrides `policy.toml [meta] profile`); values: `minimal`, `default`, `full`
- `SANDFLOX_ENABLED` — Set to `"1"` by Flox vars; indicates sandbox is active
- `SANDFLOX_MODE` — Set to `"enforced"` by Flox vars
- `PYTHON_NOPIP` — Set to `"1"` to block pip bootstrapping
- `PYTHONDONTWRITEBYTECODE` — Set to `"1"` to prevent .pyc creation
- `PYTHONPATH` — Prepended with sandflox-python cache dir for `usercustomize.py`
- `PYTHONUSERBASE` — Points to sandflox-python cache for user site customization
- `FLOX_ENV_CACHE` — Flox-provided cache directory (used for all staged enforcement artifacts)

**Policy Configuration:**
- `policy.toml` — Declarative security policy (version 2 schema)
  - `[meta]` — Policy version, default profile selection
  - `[network]` — Network mode (`blocked`/`unrestricted`), localhost allowance
  - `[filesystem]` — FS mode (`workspace`/`strict`/`permissive`), writable/read-only/denied path lists
  - `[profiles.*]` — Named presets overriding network, filesystem, and requisites

**Requisites (binary whitelists):**
- `requisites.txt` — Default profile: ~55 tools (shell utils, text processing, python3, jq, curl, git)
- `requisites-minimal.txt` — Minimal profile: ~28 tools (read-only inspection, no cp/mv/rm/curl/git)
- `requisites-full.txt` — Full profile: ~54+ tools (all non-manager binaries including find, touch)

**Build:**
- No build configuration — fully interpreted project
- `.flox/env/manifest.toml` — Flox environment definition (packages, hooks, profile scripts, vars)
- `.flox/env/manifest.lock` — Flox lockfile (auto-generated, committed)

## Platform Requirements

**Development:**
- macOS or Linux
- Flox 1.10+ installed
- Python 3.6+ (system or Flox-provided; 3.11+ enables native `tomllib`)
- macOS: `sandbox-exec` (included with macOS since 10.5)
- Linux: `bwrap` (bubblewrap) package installed for kernel enforcement

**Production:**
- Same as development — sandflox is designed to run in CI/CD and agent session environments
- No compilation, container, or deployment tooling required
- Distributed via `git clone` + `chmod +x sandflox`

  <action>
1. Create directory `.flox/pkgs/` if it does not exist.

2. Create `.flox/pkgs/sandflox.nix`:
   ```nix
   { buildGoModule, lib }:

   buildGoModule {
     pname = "sandflox";
     version = "0.1.0";

     src = lib.fileset.toSource {
       root = ../../.;
       fileset = lib.fileset.unions [
         (lib.fileset.fileFilter (file: lib.hasSuffix ".go" file.name) ../../.)
         ../../go.mod
       ];
     };

     vendorHash = null;  # zero external dependencies (CORE-01)

     CGO_ENABLED = "0";  # static binary, no C dependencies

     buildFlags = [ "-trimpath" ];  # no build path leaks (per Open Question 2)

     ldflags = [
       "-s" "-w"
       "-X main.Version=0.1.0"
     ];

     # Skip tests during Nix build -- they run via `go test` in development
     doCheck = false;

     meta = with lib; {
       description = "macOS-native sandbox for AI coding agents";
       license = licenses.mit;
       platforms = platforms.darwin;
     };
   }
   ```

   Key decisions:
   - `vendorHash = null` because there are zero external Go dependencies (Pitfall 1)
   - `lib.fileset.toSource` includes only `.go` files and `go.mod` -- NOT `policy.toml` or `requisites*.txt` which are runtime config, not build inputs (Pitfall 6)
   - `CGO_ENABLED = "0"` for a static binary (no cgo needed)
   - `-trimpath` as a build flag (not ldflag) per Open Question 2 from RESEARCH.md
   - `-s -w` strips debug info and DWARF symbols for smaller binary
   - `-X main.Version=0.1.0` injects version at build time
   - `doCheck = false` skips `go test` during Nix build (tests are run separately in development; the real `policy.toml` file is not available in the Nix build sandbox)
   - `platforms = platforms.darwin` since sandflox is macOS-only

3. Ensure all Go source files and go.mod are git-tracked (Pitfall 8):
   ```bash
   git add *.go go.mod .flox/pkgs/sandflox.nix
   ```

4. Run `flox build` and verify:
   - Build completes without errors
   - Binary exists at `result-sandflox/bin/sandflox` (or `result/bin/sandflox`)
   - Binary is executable
   - Binary runs: `./result-sandflox/bin/sandflox --debug --policy policy.toml 2>&1` shows `[sandflox] Profile:` line

5. If `flox build` fails:
   - Check for missing git-tracked files (Pitfall 8)
   - Check for vendorHash issues (Pitfall 1)
   - Check for `-trimpath` location (Open Question 2) -- may need to be in `buildFlags` or removed from ldflags
   - Read the build log carefully and fix the Nix expression

6. Verify the binary version: `./result-sandflox/bin/sandflox --version` or check binary strings for "0.1.0" (if --version not implemented, just verify the build succeeds and binary runs).

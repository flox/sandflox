  <action>
1. Create `policy.go` with `package main` containing:

   **Types** (per D-01 and research Pattern 1):
   ```go
   type Policy struct {
       Meta       MetaSection
       Network    NetworkSection
       Filesystem FilesystemSection
       Profiles   map[string]ProfileSection
   }
   type MetaSection struct {
       Version string
       Profile string
   }
   type NetworkSection struct {
       Mode           string // "blocked" | "unrestricted"
       AllowLocalhost bool
   }
   type FilesystemSection struct {
       Mode     string   // "permissive" | "workspace" | "strict"
       Writable []string
       ReadOnly []string
       Denied   []string
   }
   type ProfileSection struct {
       Requisites string
       Network    string
       Filesystem string
   }
   ```

   **ParsePolicy(path string) (*Policy, error)** function (~200 lines per D-01):
   - Read file line by line using `bufio.Scanner`
   - Track current section path as `[]string` (e.g., `["profiles", "minimal"]`)
   - For each non-blank, non-comment line:
     a. Match section header `[section]` or `[dotted.section]` via string parsing
     b. Match key = value pairs by splitting on first `=`
     c. Parse values: quoted strings (find closing quote, ignore trailing comment per Pitfall 2), booleans (`true`/`false`), string arrays (`["a", "b"]`)
     d. Reject unsupported TOML: `[[` array-of-tables, `{` inline tables, multiline strings `'''`/`"""`
   - Map parsed key-value pairs into typed struct fields using the section path
   - Handle kebab-case keys: `allow-localhost` -> AllowLocalhost, `read-only` -> ReadOnly

   **Validation** (per D-02, D-03, and CONCERNS.md schema validation gap):
   - Hard error if `Meta.Version != "2"` with message `[sandflox] ERROR: unsupported policy version "X" (expected "2")`
   - Validate `Network.Mode` against `"blocked"` and `"unrestricted"` -- error: `[sandflox] ERROR: invalid network mode "X" (expected "blocked" or "unrestricted")`
   - Validate `Filesystem.Mode` against `"permissive"`, `"workspace"`, `"strict"` -- error: `[sandflox] ERROR: invalid filesystem mode "X" (expected "permissive", "workspace", or "strict")`
   - Validate each `ProfileSection.Network` (if non-empty) against same network mode enum
   - Validate each `ProfileSection.Filesystem` (if non-empty) against same filesystem mode enum
   - All errors include `[sandflox] ERROR:` prefix per CORE-07 convention

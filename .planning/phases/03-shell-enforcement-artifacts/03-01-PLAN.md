      2. `_sfx_check_write_target` function (ports manifest.toml.v2-bash:170-204). Structure:
         - Takes `$1` as target, resolves to absolute path via `cd $(dirname)/pwd + basename` trick (verbatim from bash).
         - For EACH `{{range .Denied}}`: emit `case "$resolved" in {{. | shellquote}}*) echo "[sandflox] BLOCKED: write to {{.}} is denied by policy" >&2; return 126;; esac`.

To fix, every case pattern `{{. | shellquote}}*` MUST be emitted as TWO alternatives: `{{. | shellquote}}|{{. | shellquote}}/*`. This matches exact path OR path-plus-trailing-slash-and-subpath, without over-matching similar-prefixed paths. This applies to all three case blocks (denied/writable/readonly).
